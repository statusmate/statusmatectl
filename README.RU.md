# st4 — CLI для Statusmate

🌐 [English](README.md) · **Русский**

`st4` — консольная утилита для управления статус-страницами [Statusmate](https://statusmate.top):
инцидентами, обслуживаниями, компонентами, командами и подписчиками.

## Установка

### Homebrew (macOS / Linux)

```bash
brew install statusmate/tap/st4
```

Если tap ещё не подключён, можно сделать это явно:

```bash
brew tap statusmate/tap
brew install st4
```

Проверить установку:

```bash
st4 version
```

### Обновление через Homebrew

```bash
brew update
brew upgrade st4
```

Обновить только формулу без обновления остальных пакетов:

```bash
brew upgrade statusmate/tap/st4
```

### Сборка из исходников

```bash
go build -o st4 .
```

## Быстрый старт

```bash
# Авторизация (по умолчанию используется сервер statusmate.top)
st4 login

# Выбрать статус-страницу по умолчанию
st4 config use-status-page

# Посмотреть, под кем вы вошли
st4 whoami

# Список статус-страниц
st4 ls

# Список инцидентов / компонентов / обслуживаний
st4 ls i
st4 ls c
st4 ls m

# Создать инцидент / обслуживание
st4 create-incident --help
st4 create-maintenance --help
```

## Конфигурация

`st4` хранит конфигурацию **отдельно для каждого сервера**. Это позволяет работать
с несколькими инсталляциями Statusmate (например, `statusmate.top` и собственный
self-hosted сервер) без переключения профилей вручную.

### Где лежит конфиг

Файл `authrc` в формате JSON находится по пути:

```
~/.st4/<домен-сервера>/authrc
```

Например, для сервера по умолчанию `statusmate.top`:

```
~/.st4/statusmate.top/authrc
```

Узнать точный путь к конфигу:

```bash
st4 config path
```

Посмотреть текущее содержимое конфига:

```bash
st4 config show
```

### Что хранится в конфиге

| Поле                   | Описание                                           |
| ---------------------- | -------------------------------------------------- |
| `token`                | Токен авторизации (создаётся командой `st4 login`) |
| `default_status_page`  | Статус-страница по умолчанию                       |
| `default_release_page` | Страница релизов по умолчанию                      |
| `default_team`         | Команда по умолчанию                               |
| `api`                  | Адрес API сервера                                  |

Значения по умолчанию подставляются в команды автоматически, поэтому не нужно
каждый раз указывать `--status-page` или `--team`.

### Выбор значений по умолчанию

```bash
# Выбрать статус-страницу по умолчанию (интерактивный список)
st4 config use-status-page

# Показать текущую статус-страницу по умолчанию
st4 config current-page

# Выбрать команду по умолчанию (интерактивный список)
st4 config use-team

# Показать текущую команду по умолчанию
st4 config current-team
```

### Выбор сервера

По умолчанию используется `statusmate.top`. Сервер можно переопределить флагом
`--server` или переменной окружения `ST4_SERVER`:

```bash
# Через флаг
st4 --server=statusmate.top login

# Через переменную окружения
export ST4_SERVER=statusmate.top
st4 login
```

Для каждого сервера ведётся свой каталог конфигурации в `~/.st4/`, поэтому
авторизация и настройки по умолчанию не пересекаются между серверами.

## Основные команды

```bash
st4 login                 # авторизация (поддерживается 2FA)
st4 whoami                # информация о текущем пользователе
st4 ls                    # список статус-страниц
st4 ls i                  # список инцидентов
st4 ls c                  # список компонентов
st4 ls m                  # список обслуживаний
st4 ls t                  # список команд
st4 ls u                  # список участников команды
st4 create-incident       # создать инцидент
st4 create-maintenance    # запланировать обслуживание
st4 status                # дерево статусов компонентов
st4 tui                   # интерактивный TUI-дашборд
st4 open                  # открыть статус-страницу в браузере
st4 config ...            # управление конфигурацией (см. выше)
st4 version               # версия утилиты
```

Список всех команд и флагов:

```bash
st4 --help
st4 <command> --help
```
